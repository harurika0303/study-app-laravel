# ADR-0005: UI スタイリングに Tailwind CSS を採用する

## ステータス
承認済み

## 背景
フロントエンドの UI 実装方針として、CSS フレームワーク / コンポーネントライブラリの選定を行う必要があった。
Vue 3 + Inertia.js の構成で、カスタマイズ性・長期保守性が重視される。

## 決定
**Tailwind CSS v3** を採用する。UI コンポーネントライブラリは使用せず、Tailwind のユーティリティクラスで自前コンポーネントを実装する。

## 代替案

| 案 | 却下理由 |
|----|---------|
| Vuetify | Material Design に縛られる。Tailwind との共存が難しい |
| shadcn/ui（React 専用）| Vue では使用不可 |
| PrimeVue | デフォルトスタイルのカスタマイズに手間がかかる |
| Bootstrap | デフォルト UI が古臭く見える。Tailwind のユーティリティファーストより柔軟性が低い |

## 結果
- **ポジティブ**: Tailwind の JIT コンパイルで未使用 CSS を完全排除できる。`prettier-plugin-tailwindcss` でクラス順序を自動整列できる。デザインに完全な自由度がある。
- **ネガティブ**: 再利用コンポーネントを自前で実装する必要があり、初期実装コストが高い。
