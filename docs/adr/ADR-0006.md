# ADR-0006: Vue コンポーネントは Composition API（`<script setup>`）のみで実装する

## ステータス
承認済み

## 背景
Vue 3 では Options API と Composition API の両方が使用可能だが、
方針が統一されていないと、コントリビューターごとにスタイルが混在しコードの一貫性が失われる。

## 決定
**すべての Vue コンポーネントで `<script setup lang="ts">` を使用する。** Options API の使用を禁止する。

## 理由
- `<script setup>` はボイラープレートが少なく、型推論が自然に効く
- TypeScript との親和性が高い（`defineProps<T>()` 等）
- Vue 3 の公式推奨スタイル
- Composable（`useXxx`）による再利用ロジックの分離が容易

## ルール
- `<script setup lang="ts">` を必ず付与する
- Composable は `resources/js/composables/` に配置し、`use` プレフィックスを付ける
- Props は `defineProps<{ ... }>()` でインターフェースを使って型定義する
- Emits は `defineEmits<{ eventName: [payload: Type] }>()` で型定義する

## 結果
- **ポジティブ**: コードの一貫性が保たれ、レビューコストが下がる。型安全な Props/Emits を強制できる。
- **ネガティブ**: Vue 2 や Options API に慣れた開発者には学習コストがある。
