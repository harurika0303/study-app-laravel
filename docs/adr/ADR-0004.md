# ADR-0004: セルフホスト対応のため Docker Compose ベースの構成を採用する

## ステータス
承認済み

## 背景
デプロイ先として将来的に自前のラズパイ（Raspberry Pi）を使う可能性が示された。
「docker compose up` 一発で本番環境を構築できる構成が求められた。
Laravel + MySQL + Redis + Nginx を一括管理できる構成が必要だった。

## 決定
**Docker Compose** でサービスを定義し、`docker-compose.yml`（開発）と `docker-compose.prod.yml`（本番）を分離する。

### サービス構成
| サービス | イメージ | 役割 |
|---------|---------|------|
| `app` | PHP 8.3-FPM Alpine カスタム | Laravel 本体 |
| `nginx` | nginx:1.25-alpine | Web サーバー |
| `db` | mysql:8.0 | データベース |
| `redis` | redis:7-alpine | セッション・キャッシュ |
| `vite` | node:20-alpine | 開発時 HMR（開発環境のみ）|

## 代替案

| 案 | 却下理由 |
|----|---------|
| Laravel Sail（Docker ラッパー）| ARM（Raspberry Pi）では一部イメージが非対応。カスタマイズ性が低い |
| 直接インストール（non-Docker）| 環境差異が生じやすく、再現性が保てない |

## 結果
- **ポジティブ**: `docker compose up -d --build` 一発でどの環境でも同一構成が起動する。ARM（Raspberry Pi 4+）にも Alpine ベースイメージで対応可能。
- **ネガティブ**: Docker 未経験者には初期学習コストがある。
