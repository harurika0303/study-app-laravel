# ADR-0007: データ取得は Controller（Inertia::render）・更新は router.patch/post/delete で行う

## ステータス
承認済み

## 背景
Inertia.js を使用する場合、データ取得とミューテーションの実装方法として複数の選択肢が存在する。
方針が統一されていないと、同じ操作が複数の経路で実装され一貫性が失われる。

## 決定
- **データ取得**: Controller の `Inertia::render()` でデータを渡す。クライアントからの API 呼び出し（axios 等）でのデータ取得は原則禁止
- **データ更新**: Inertia の `useForm` または `router.patch/post/delete` を使用する。REST API エンドポイントを別途作成しない
- **例外**: タグの即時作成（ADR-0017）のみ `axios.post` を許可する

## 理由
- Inertia のサーバーサイドルーティングを活かし、Laravel の認証・CSRF・バリデーションをそのまま利用できる
- REST API レイヤーを作らないことで実装を一本化できる
- `Inertia::render()` に渡すデータはサーバーで完結するため、型定義が PHP + TypeScript Resource で二重管理できる

## 結果
- **ポジティブ**: 実装の経路が統一され、コードが追いやすくなる。API 設計の意思決定が不要になる。
- **ネガティブ**: 将来的にモバイルアプリから同じデータを使いたい場合は REST API を別途追加する必要がある。
