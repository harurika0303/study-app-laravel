# ADR-0011: 日付・タイムゾーン処理はブラウザのローカルタイムゾーンに合わせる

## ステータス
承認済み

## 背景
DB は UTC で `DATETIME` を保存している（MySQL のデフォルト）。
ダッシュボードの「今週の学習時間」「カレンダーヒートマップ」では「今日」「今週」の境界を
どのタイムゾーンで判断するかが問題になる。
ADR-0003 でマルチユーザー対応設計を採用しており、日本以外のユーザーも想定される。

## 決定
1. ブラウザで `Intl.DateTimeFormat().resolvedOptions().timeZone` を取得し、Cookie `tz` にセットする（`AppLayout.vue` の `onMounted` で実行）
2. Controller では `$request->cookie('tz', 'Asia/Tokyo')` でタイムゾーンを読み取り、集計クエリに適用する
3. MySQL の `CONVERT_TZ()` または Carbon の `setTimezone()` を使ってローカル日付に変換する

## ルール
- DB への保存は常に UTC（Laravel の `DATETIME` カラムは UTC として扱う）
- 集計・表示時のみタイムゾーン変換を行う
- フロントエンドへのレスポンスは ISO 8601 UTC 文字列（`toISOString()`）で返し、表示時にブラウザのロケールで変換する

## 結果
- **ポジティブ**: 日本以外のユーザーでもカレンダーの日付がずれない。
- **ネガティブ**: Cookie が存在しない初回リクエスト時はデフォルトの `Asia/Tokyo` を使う。
