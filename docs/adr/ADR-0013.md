# ADR-0013: セッション操作は削除のみとし、編集機能を提供しない

## ステータス
承認済み

## 背景
学習セッションは「いつからいつまで学習したか」という過去の事実を記録するものであり、
誤入力を修正したい場合の操作方法を設計する必要があった。

## 決定
セッションの**削除**のみ可能とする。編集機能は提供しない。

誤ったセッションを記録した場合は削除して再記録する。

## 理由
- 編集機能を提供すると「過去の記録を改ざん」する操作が可能になり、記録の信頼性が下がる
- 削除 → 再記録で同等の訂正が可能であり、UX 上の問題は少ない
- 実装コストの削減（編集フォーム・バリデーション・ルーティングが不要）

## 影響
- `SessionController` は `store` と `destroy` のみを実装する
- `routes/web.php` でも対応するルートのみ定義する
