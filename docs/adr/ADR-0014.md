# ADR-0014: 最初のセッション記録時に NOT_STARTED → IN_PROGRESS へ自動遷移する

## ステータス
承認済み

## 背景
コンテンツを追加した直後のステータスは `NOT_STARTED`（未着手）である。
ユーザーが学習を開始してセッションを記録したとき、ステータスをどう扱うかを決定する必要があった。

## 決定
`SessionController::store` 内で、コンテンツの現在のステータスが `NOT_STARTED` の場合のみ、
自動的に `IN_PROGRESS` に更新する。

```php
// app/Http/Controllers/SessionController.php
public function store(StoreSessionRequest $request, LearningContent $content): RedirectResponse
{
    $this->authorize('create', [LearningSession::class, $content]);

    DB::transaction(function () use ($request, $content) {
        $content->sessions()->create([
            'user_id'    => $request->user()->id,
            'started_at' => $request->validated('started_at'),
            'ended_at'   => $request->validated('ended_at'),
            'memo'       => $request->validated('memo'),
        ]);

        // NOT_STARTED の場合のみ自動遷移
        if ($content->status === ContentStatus::NotStarted) {
            $content->update(['status' => ContentStatus::InProgress]);
        }
    });

    return back()->with('success', 'セッションを記録しました');
}
```

## 理由
- ユーザーが「学習を始めた」という行為（セッション記録）を、ステータス変更という別操作なしに自動反映できる
- `IN_PROGRESS` → `COMPLETED/PASSED/FAILED` への遷移は意図的な操作であるため手動のままとする

## 影響
- `SessionController::store` にステータス自動遷移ロジックを追加する
- 自動遷移は `NOT_STARTED → IN_PROGRESS` のみ。それ以外のステータスには影響しない
