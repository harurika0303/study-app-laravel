# ADR-0008: グローバル状態管理は Inertia の shared data に限定し、Pinia は使用しない

## ステータス
承認済み

## 背景
Vue 3 では Pinia 等の外部状態管理ライブラリが選択肢として存在するが、
データの大半をサーバーサイド（Controller）で取得する設計（ADR-0007）を選択した。
その上でクライアントサイドのグローバル状態管理をどの範囲・手段で行うかを決める必要があった。

## 決定
グローバル状態管理として **Inertia の shared data**（`HandleInertiaRequests::share()`）のみを使用する。
Pinia 等の外部状態管理ライブラリは導入しない。

shared data で管理するもの:
- 認証ユーザー情報（`auth.user`）
- フラッシュメッセージ（`flash.success`, `flash.error`）

## 理由
- 大半のデータはサーバーサイドでレンダリングされるため、クライアントサイドの状態管理ニーズが少ない
- タイマー等のローカル状態は Composable（`useSessionTimer`）+ `localStorage` で十分管理できる
- Pinia を導入すると状態の「どこで管理するか」の判断コストが増える

## 結果
- **ポジティブ**: 状態管理の経路が単純化される。Pinia の学習・管理コストが不要。
- **ネガティブ**: 複雑なクライアントサイド状態が必要になった場合に後から Pinia を追加するコストがかかる。
