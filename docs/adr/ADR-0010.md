# ADR-0010: ミューテーション後のページ更新に Inertia の自動再レンダリングを使用する

## ステータス
承認済み

## 背景
Inertia.js を使用する場合、フォーム送信後にデータを最新の状態に更新する方法を決める必要があった。
Next.js の `revalidatePath` に相当する仕組みが必要だった。

## 決定
Inertia の標準動作を利用する。
`router.post/patch/delete` でフォームを送信すると、サーバーがリダイレクトまたは同じページを返し、
クライアントは自動的に最新データで再レンダリングされる。

追加のキャッシュ無効化操作は不要。

## ルール
- 作成・更新・削除後は `redirect()->route('xxx')` で適切なページにリダイレクトする
- 即時更新（インライン編集等）は `router.patch()` を使い、レスポンスで同ページを再レンダリングする
- `preserveScroll: true` オプションを使うと、ページスクロール位置を保持したまま更新できる

```typescript
// ✅ ステータスのインライン更新
router.patch(route('contents.status', props.content.id), { status: newStatus }, {
    preserveScroll: true,
});
```

## 結果
- **ポジティブ**: Next.js の `revalidatePath` のような手動キャッシュ無効化が不要。Inertia が自動的に処理する。
- **ネガティブ**: 特になし。
