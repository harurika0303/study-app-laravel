# ADR-0003: 個人利用を主目的としながらマルチユーザー構成を採用する

## ステータス
承認済み

## 背景
アプリの主な利用者は「自分1人（個人用）」と明示された一方、認証・ユーザー管理の方針として
「ログインあり・複数ユーザー対応」が選択された。
この組み合わせは一見矛盾するが、**将来的に他者へ公開・共有する可能性を残したい意図**と解釈し、
シングルユーザーで始めながらマルチユーザーに対応できる設計を最初から採用するという決定を下した。

## 決定
全テーブルに `user_id` 外部キーを持たせ、Controller・Policy で必ず `user_id = auth()->id()` を条件に加える。
ユーザー間のデータは完全に分離する。

## 結果
- **ポジティブ**: 将来の一般公開に対応できる。シングルユーザー運用時もセキュリティが確保される。
- **ネガティブ**: クエリに常に `where('user_id', $user->id)` が必要になるが、Eloquent の scope として定義することで負担を最小化できる。
- **制約**: 本 ADR を覆す場合、全テーブルから `user_id` を除去し、全 Controller・Policy を再実装する必要がある。
