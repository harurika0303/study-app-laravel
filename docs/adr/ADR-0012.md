# ADR-0012: コンテンツのサムネイル画像は Laravel Storage（ローカルディスク）に保存する

## ステータス
承認済み

## 背景
学習コンテンツ（書籍の書影・動画コースのサムネイル等）に画像を付けたい要件が挙がった。
画像の保存先として複数の選択肢を検討した。
Laravel の標準機能として Filesystem（Storage）が利用可能であり、追加インフラ不要での実装が可能だった。

## 決定
**Laravel Storage（`public` ディスク、`storage/app/public/thumbnails/`）** を使用する。

- 開発環境・自己ホスト環境: ローカルディスク
- クラウドへの移行が必要になった場合: `.env` の `FILESYSTEM_DISK=s3` を変更するだけで S3 互換ストレージに移行可能（`Storage::url()` の呼び出し箇所は変更不要）

## アップロードフロー
1. `ContentController::store` / `update` で `$request->file('thumbnail')->store('thumbnails', 'public')` を実行
2. 戻り値のパス（例: `thumbnails/xxxx.jpg`）を `thumbnail_path` カラムに保存する
3. 公開 URL は `Storage::url($content->thumbnail_path)` で生成し、`ContentResource` で返す
4. 既存サムネイルの更新時は古いファイルを `Storage::delete($oldPath)` で削除する

## 結果
- **ポジティブ**: 追加インフラ不要。S3 互換ストレージへの移行が `.env` 変更だけで対応可能。
- **ネガティブ**: 自己ホスト環境では定期バックアップを自前で設定する必要がある。
