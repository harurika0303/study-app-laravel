# ADR-0015: インポート機能をスコープ外とする

## ステータス
承認済み

## 背景
エクスポート（JSON/CSV）と対になるインポート機能（JSON アップロード → DB 挿入）を検討した。
実装するには以下のコストが発生する:

- JSON パース・バリデーション・重複チェックのロジック
- 複数テーブルへのバルク INSERT トランザクション
- エラー件数・スキップ件数のレスポンス設計

## 決定
**インポート機能はスコープ外**とする。エクスポート機能のみ実装する。

## 理由
- 個人利用が主目的であり、初期データ移行は稀なユースケース
- バリデーション・重複チェックの設計コストに見合う価値が現時点では低い
- 将来必要になった場合に追加実装する（`ImportController` + Job キューで非同期処理）

## 影響
- `settings` ページの「データ管理」タブにはエクスポートボタンのみを表示する
- インポートボタン・UI・API エンドポイントは実装しない
